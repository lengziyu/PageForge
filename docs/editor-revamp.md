# 编辑器改版草图

目标：把当前“直接堆配置表单”的体验，改成“先理解页面结构，再进入具体配置”。

## 页面编辑器草图

```text
+----------------------------------------------------------------------------------+
| 顶部操作栏：页面标题 / 状态 / 返回 / 内容中心 / 预览 / 保存草稿 / 发布整站        |
+----------------------------------------------------------------------------------+
| 左栏：结构视图 / 添加模块  | 中间：画布预览                                      |
|---------------------------|----------------------------------------------------|
| 页面概览                  | 顶部导航预览                                       |
| - 当前页信息              | [模块 1] Hero                                      |
| - 打开全站设置            | [模块 2] 产品展示                                  |
| - 继续加模块              | [模块 3] 新闻列表                                  |
|                           | [模块 4] CTA                                       |
| 模块大纲                  | 页脚预览                                           |
| 1. Hero                   |                                                    |
| 2. 产品展示 (产品中心)    |                                                    |
| 3. 新闻资讯 (新闻中心)    |                                                    |
| 4. CTA                    |                                                    |
+---------------------------+-----------------------------+----------------------+
| 右栏：模块设置 / 公共设置 | 内容                         | 样式 / 布局          |
|---------------------------|-----------------------------|----------------------|
| 模块信息                  | 标题、描述、数据源           | 样式变体、数量、间距 |
| 空状态提示                | 列表项 / 动态源说明          | 背景、容器宽度       |
+----------------------------------------------------------------------------------+
```

## 配置分组建议

- `内容`：标题、描述、按钮文案、列表项、数据源。
- `样式`：展示变体、卡片风格、背景、强调色。
- `布局`：列数、间距、容器宽度、对齐方式。
- `全站设置`：站点信息、导航、页脚、品牌资源。

## 动态内容设计

- 新闻模块：`手动维护 / 新闻中心`
- 产品模块：`手动维护 / 产品中心`
- 后续可扩展：案例中心、解决方案中心、招聘中心
